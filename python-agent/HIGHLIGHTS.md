# Python Agent 模块 — 项目亮点（简历用）

> **技术栈：** Python, FastAPI, DeepSeek/OpenAI SDK, RAG, Function Calling, asyncio, Pydantic
> **对接后端：** Spring Boot, MySQL, Redis, RabbitMQ, JWT

---

## 核心亮点

### 1. RAG + Function Calling 智能对话引擎

用户自然语言输入后，系统并发执行三层检索（公共词库查词、个人单词本匹配、用户学习概况），通过 `asyncio.gather` 将串行 3+N 次 API 调用压缩为 3 个并发协程。检索结果送入 DeepSeek 进行 Function Calling，LLM 自动拆解指令并编排最多 3 轮工具调用。模块级停用词过滤 + 60s 内存缓存，上下文构建延迟 < 50ms。

对比同类实习项目：多数 RAG 实现仅做单层向量检索，本项目增加了用户个性化数据（积分、单词本、学习进度）的多源融合，且结合 Function Calling 实现了闭环操作能力。

### 2. 多层次降级保障

| 故障场景 | 降级策略 |
|---------|---------|
| LLM 不可用 | 本地关键词匹配回复（零网络依赖） |
| 后端 API 异常 | 返回友好提示，服务不崩溃 |
| Token 过期 | 自动清除并提示重新登录 |

三层降级全覆盖，任何单一依赖故障不影响用户可用。对比 LangChain Agent 等框架在 LLM 不可用时直接报错，本方案在工程健壮性上有明显提升。

### 3. 模块化工具热注册体系

```
新增一个业务工具只需三步：
endpoints.py 添加 API 封装 → tools.py 注册 Tool 定义 → (可选) RAG 上下文
```

无需修改 LLM 调用逻辑、路由分发或消息格式。单词查询、每日签到、积分管理、商店浏览、秒杀抢购等 8 个工具已在线运行，单工具平均 5 行注册代码。对比 LangChain 的工具注册流程（需定义 Schema + 参数校验 + 错误处理），本方案更轻量。

### 4. 对话生命周期管理

多用户对话独立隔离，消息总量超过 15 条时自动对历史做抽取式摘要（保留最近 10 条完整对话），启动时清理 7 天前的过期数据。首条消息自动检测话题标签（单词查询 / 积分查询 / 秒杀活动等 8 类），实现不同场景的语义路由。

### 5. JWT Token 透传认证

前端登录后将 JWT Token 通过请求注入 Agent，自动附加到所有后端 API 调用。支持 session 本地持久化，同一终端多次请求不需要重复登录。

## 对比同类项目

| 维度 | 本项目 | 典型实习项目（如 Agent Craft / 简历问答Bot） |
|------|--------|---------------------------------------------|
| RAG 数据源 | 公共词库 + 用户个人单词本 + 学习概况（多源融合） | 单文档/单知识库向量检索 |
| 工具调用 | 8 个真实业务工具，3 轮链式编排 | 通常 1-2 个演示工具 |
| 后端集成 | 对接真实 Spring Boot 生产系统 | 通常独立运行或 Mock 数据 |
| 降级策略 | 三层降级（LLM/后端/Token） | 通常无降级 |
| 并发优化 | asyncio.gather 并发协程 | 通常串行调用 |
| 对话管理 | 自动摘要 + 过期清理 + 话题检测 | 通常无或基础 |

---

## 解决的问题

1. **AI 不编造数据** — 所有操作经后端 API 执行，积分、签到、单词都是真实数据
2. **复合指令** — "查积分并签到"自动拆解为多步工具调用链
3. **用户感知** — RAG 检索用户个性化数据，回复不泛泛而谈
4. **工程健壮** — 任何外部依赖故障都能优雅降级
