# 后端新增模块记录

## 1. 慢查询监控模块 (DbTimeAspect)
**文件：**
- `aspect/DbTimeAspect.java` — AOP 切面
- `entity/SlowQueryLog.java` — 慢查询日志实体
- `mapper/SlowQueryLogMapper.java` — MyBatis Mapper
- `mapper/SlowQueryLogMapper.xml` — XML 映射（待补）
- `db/init_slow_query_log.sql` — 建表 SQL（待补）

**功能：**
自动拦截所有 Mapper 方法，统计 SQL 执行耗时：
- > 200ms → 控制台黄色警告
- > 500ms → 红色报警 + 自动写入 slow_query_log 表
- 排除 SlowQueryLogMapper 自身，避免递归

## 2. 分布式限流模块 (RateLimiter)
**文件：**
- `utils/RateLimiter.java` — 限流器
- `exception/RateLimitException.java` — 限流异常
- `exception/GlobalExceptionHandler.java` — 全局异常处理（含限流返回）
- `resources/lua/token_bucket.lua` — 令牌桶 Lua 脚本
- `resources/lua/fixed_window.lua` — 固定窗口 Lua 脚本

**功能：**
两层限流保护秒杀接口：
- 用户级：固定窗口，每秒最多 3 次
- 全局级：令牌桶，容量 300，每秒补 200

**注意：** `pom.xml` 需包含 `**/*.lua` 到 resources 编译配置（已修复）。

---

## 3. 秒杀库存定时同步 (SeckillStockSyncTask)
**文件：**
- `task/SeckillStockSyncTask.java`

**功能：**
4 个定时任务保障 Redis 和 MySQL 数据一致：
- 每小时整点：MySQL → Redis 同步秒杀库存
- 每分钟：刚结束的秒杀活动强制同步一次
- 每 30 秒：MQ 失败消息扫表重试（最多 5 次）
- 每分钟：超时未处理订单补偿重试

**启动时：** `@PostConstruct` 自动预热秒杀库存到 Redis。

---

## 4. RabbitMQ 死信队列 (DLQ)
**文件：**
- `config/RabbitMQConfig.java` — DLX/DLQ 配置 + 业务队列绑定
- `mq/consumer/DlqConsumer.java` — 死信消费者

**功能：**
- 每个业务队列绑定死信交换机
- 消息重试 3 次失败 → 自动投递到 DLQ
- 人工介入处理死信消息
